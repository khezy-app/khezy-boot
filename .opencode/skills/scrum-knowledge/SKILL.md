## SCRUM Framework Knowledge

When asked about Scrum, Agile, or related methodologies, load this skill and reference the theory documents at `scrum/theory/SCRUM_THEORY.md` (Scrum) and `scrum/theory/AGILE_THEORY.md` (Agile) for comprehensive answers.

### How to answer Scrum/Agile questions

1. **Distinguish Scrum from Agile**
   - **Agile** is a broad philosophy defined by the [Agile Manifesto](https://agilemanifesto.org/) (2001): individuals & interactions over processes & tools, working software over comprehensive documentation, customer collaboration over contract negotiation, responding to change over following a plan.
   - **Scrum** is a specific framework that implements Agile principles. It's the most widely adopted Agile framework.

2. **Core Scrum structure** (per the 2020 Scrum Guide)
   - **3 accountabilities**: Product Owner, Scrum Master, Developers
   - **5 events**: Sprint, Sprint Planning, Daily Scrum, Sprint Review, Sprint Retrospective
   - **3 artifacts**: Product Backlog, Sprint Backlog, Increment
   - **3 commitments**: Product Goal, Sprint Goal, Definition of Done
   - **3 pillars**: Transparency, Inspection, Adaptation
   - **5 values**: Commitment, Focus, Openness, Respect, Courage

3. **Common misconceptions to address**
   - Scrum is not a methodology — it's a lightweight framework
   - The Daily Scrum is not a status report for management
   - The Product Owner is not a project manager
   - The Scrum Master is not a team lead or manager
   - Sprints are not necessarily release cycles

4. **When the user asks about specific topics**
   - "How to start Scrum?" → Explain the three accountabilities, timeboxed Sprint, and minimal artifacts
   - "Scrum vs Kanban?" → Scrum uses fixed-length Sprints with prescribed roles/events; Kanban is continuous flow without prescribed roles
   - "What is a good Sprint length?" → 1-2 weeks is most common for software teams; ≤1 month per Scrum Guide
   - "Can we change Scrum?" → The framework is immutable; partial adoption is not Scrum
   - "Scrum with remote teams?" → All events work remotely; Daily Scrum and Sprint Review benefit from good video/board tooling
   - "How to estimate?" → Scrum doesn't prescribe estimation techniques; common: story points, t-shirt sizing, ideal hours
   - "What is Definition of Done?" → A formal quality standard; if an item doesn't meet it, it's not an Increment

5. **Always reference the source**
   - The Scrum Guide 2020 is the authoritative definition
   - Link to https://scrumguides.org/scrum-guide.html when appropriate
