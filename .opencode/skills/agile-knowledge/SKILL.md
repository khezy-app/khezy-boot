## AGILE Methodology Knowledge

When asked about Agile, the Agile Manifesto, or related practices, load this skill and reference the theory document at `scrum/theory/AGILE_THEORY.md` for comprehensive answers.

### How to answer Agile questions

1. **Start with the Agile Manifesto** (2001)
   - 4 value statements: individuals & interactions, working software, customer collaboration, responding to change
   - These are the foundation of all Agile frameworks

2. **Explain the 12 Principles** — they define what makes a method "Agile"
   - Customer satisfaction, welcome change, frequent delivery, collaboration, motivated teams, face-to-face comms, working software, sustainable pace, technical excellence, simplicity, self-organizing teams, continuous improvement

3. **Distinguish Agile from its frameworks**
   - **Agile** = philosophy/principles
   - **Scrum** = framework with Sprints, roles, events (most popular)
   - **Kanban** = continuous flow, visual boards, WIP limits
   - **XP** = technical practices (TDD, pair programming, CI)
   - See `scrum/theory/AGILE_THEORY.md` §5 for full list

4. **Contrast with Waterfall** when asked
   - Waterfall: sequential phases, fixed requirements upfront, single delivery
   - Agile: iterative cycles, emergent requirements, continuous delivery

5. **Common misconceptions to address**
   - Agile ≠ no planning (it plans continuously, adaptively)
   - Agile ≠ no documentation (just enough docs, working software is priority)
   - Agile ≠ Scrum (Scrum is one framework under the umbrella)
   - Agile ≠ faster delivery (it enables faster feedback; actual speed varies)
   - Agile ≠ cowboy coding (it demands technical excellence and sustainable pace)

6. **When the user asks about specific topics**
   - "Agile vs Waterfall?" → Iterative vs sequential; emergent vs fixed requirements; continuous vs limited customer involvement
   - "Which Agile framework should I use?" → Scrum for structured roles/events; Kanban for continuous flow; XP for engineering rigor; depends on team context
   - "How to start with Agile?" → Pick a framework (Scrum is most common), start with one team, iterate
   - "Agile with remote teams?" → Works well with good tooling (Jira, Trello, Slack); Daily stand-ups and retrospectives can be async
   - "Is Agile only for software?" → Originated in software but now used in marketing, HR, hardware, education
   - "What are Agile ceremonies?" → Stand-ups, sprint planning, reviews, retrospectives (specifics depend on framework)
   - "How to estimate in Agile?" → Story points, t-shirt sizing, planning poker; focus on relative sizing, not hours

7. **Always reference the source**
   - The Agile Manifesto (https://agilemanifesto.org/) is the authoritative foundation
   - The 12 Principles (https://agilemanifesto.org/principles.html) define what "Agile" means
